What ~/ does in Razor Pages today

App PathBase: /myapp

1. Static asset URLs — allowlisted attributes

1a. <img src="~/images/logo.png">

Rendered:

1b. <link href="~/css/site.css"> (already in head)

1c. <script src="~/js/site.js">

1d. <a href="~/images/logo.png"> (asset target)

download logo

1e. <source srcset="~/images/logo.png 1x, ~/images/logo-2x.png 2x">

1f. <form action="~/Submit">

2. Page route targets (no static asset behind them)

2a. <a href="~/counter">

go to /counter

2b. <a href="~/Compare?x=1"> with query

self-link with query

3. Non-URL attributes (not in tag-helper allowlist)

3a. <div title="~/foo">

hover me — title is literal

3b. <div data-url="~/foo">

data-url attr is literal

4. Programmatic features

4a. @Url.Content("~/images/logo.png")

Result: /myapp/images/logo.png

4b. @Url.Content("~/counter") (route target)

Result: /myapp/counter

4c. @Assets["images/logo.png"] — not a thing in Razor Pages

Razor Pages has no built-in Assets indexer; fingerprinting flows through tag helpers (asp-append-version, link/script tag helpers since 9.0).

4d. View Component invocation

ViewComponent received: Source=~/images/logo.png, Route=~/counter
Re-emitted via <img src="@Model.Source">:
Re-emitted via <a href="@Model.Route">: link
Re-emitted via Url.Content(@Model.Route): /myapp/counter

5. Partial / shared rendering

5a. Partial uses ~/ in <img> and <a>

Partial says: to /counter title=~/foo (non-URL)
